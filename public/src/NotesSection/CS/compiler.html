1. Introduction to Compiler

1.1. High Level language

If a program contains define or include directives such as include or define it is called HLL They are closer to humans but far from machines These tags are called dpre process directives they direct the pre processor about what to do

Macros may be nested in definitions or in uses Short hand Operators

1.2. Pre processor

The preprocessor removes all the #include directives by including the files called file inclusion All the #define directives using macro expansion. It performs file inclusin, macro processing short hand opertors etc

1.3. Compiler

A compiler is a computer program that translates computer code written in one programming language into another language The name compiler is primarly used for programs that translate source code from a high level programming language to a lower level lanugage to create an executable program The reason is we are not comfortable in writing a low level language there for we write a code which is easy and then convert it into low level language.

1.4. Assembler

Its neither binary form nor high level It is an intermediate state that is a combination of machine instructions and some other useful data for execution. For every platform we will have an assembler. They are not universal since for each platform weh ave one The output of assembler is called object file. it translates assembly language to machine code

1.5. Relocateable Machine Code

It can be loaded at any point and can be run The address within the program will be in such a way that it will cooperate for the program movement

1.6. Linker and Loader

It converts the relocatbale code into absolute code and tries to run the program resulting in a running program or an error message Linker loads a variety of object files into a single file to make it executable. Then loader loads it in memory and executes it

1.7. Compiler Design

Two phases of compiler namely Analysis phasee and synthesis phase

1.7.1. Analysis Phase

  1. Lexical Analyzer

    It reads the program and convert it into lexemes A stream of lexemes into a stream of tokens Tokens are defined by regular expression which are understood by the lexical analyzer. It also removes white spaces and comments Lexeme is a words in code Lexeme with its type is called lexemes

    Then it is converted to a general code

    Float x,y,z; x=y+z*60

    x,y,z -> token -> identifer =,+,* -> token -> operators 60 -> token -> constant

    id1 = id2 + id3 * 60

  2. Syntax Analyzer

    It is sometimes called as parser It constructs the Parse/Syntax tree Uses productions of Context Free Grammer to construct the parse tree. It reads all the tokens one by one

  3. Semantic Analyzer

    It verifies the parse tree whether it is meaningful or not It furthermore produces a verified annotatated parse tree.

1.7.2. Synthesis Phase

  1. Intermediate Code generator

    It generates intemediate code, that is a form which can be readily understood by machine. We have many popular intermediate coes. Intermediatte code is converted to machine language using the last two phases which are platform dependent. Till intermediate code, it si same for every compiler out there, but after that it depends on the platform To build a new compiler we don’t need to build it from scratch. We can take the intermediate code form the already exisiting compiler and build the last two parts

  2. Code Optimiser

    It transform the code so that it consumes fewer resources and producse more speed. The meaning of the code optimizer is code being transformed is not altered Optimisation can be categorized into two types: Machine dependent and machine independent

  3. Target Code Generation

    The main purpose of target code generator is to write a code that the machine can understand and also register allocation, instructions selections etc. The output is dependent on the type of assembler. This is the final stage of compilationh

1.8. Symbol table

It is a data strucutre being used and maintained by the compiler, consists all the identifiers’ name along with their types It helps the compiler to function smoothly by finding the identifers quickly The compiler has two modules namely front and back. Front constitues of the lexical analyzer, semantic analyser, syntax analyser and intermediate code generator. and the rest are assembled to form the back end In general front end fill symbol table and back end uses it Information stored in symbol table about identifer name, type, scope, size, offset Operations on the symbol table - there are 4 operation function that can be performed on symbol table Insert lookup modify and delete

Implementation of symbol tables can be done using anyone of the data strucutre - Linear table, Binary search tree, Linked List, Hash table

1.9. Error Handler

It is an sub routine to take care of the conitnuation of compilation, even any error at any phase, i.e error handler is responsible to continue the compilation process even any error occurs at phase 1 or phase 2 or phase 3 After phase 3 If the error handler object is empty then the source code is free of error and it can be converted into target code. If the error handler object is not empty after phase 3 athen there will be some error at phase 1,2 or 3 then the error will be displayed Compiler can have 3 types of errors lexical error syntax and semantic error The error handles by compiler are known as exception and programmer is responsible to handle the exception At the time of execution also, we can get some error they are called fatal error and system admin is responsible to handle fatal erro

1.10. Phases and passes

A compiler can have many phases and passes A pass refers to the traversal of a compiler through the entire program A phase of a compiler is a distinguishable stage, which takes input from the previous stage, processes and yield output that can be used as input for the next stage. A pass can have more than one phase
