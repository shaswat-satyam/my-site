1. JavaScript

A language which is used to manage Browser and it various events. It is a procedural language. It is most used in web browsers, browser extenions It is a dynamically typed language

2. Hard Parts

2.1. Thread of Execution

Goes through the code line by line and executes each line known as thread of exection Saves data like strings and arrays so we can use the data later in the memory It includes Function calls, Function declaration

2.2. Execution Context

It is created to run the code of a function it has 2 parts - Thread of Exection and Memory

2.3. Call Stack

JavaScript Keeps a track of what function is currently running Run a function is to add to call stack Finish running the function is to remove from the call stack Whatever is on the top of the stack is the currently running function

2.6. Asynchronous Programming

JavaScript is a synchronous language. We have a single thread only which can execute one task at a time If there is a long task like fetching API our program will be stuck. To combat this we will use asynchronous programming.

2.6.1. setTimeout

Takes a function and a delay in ms. Till then the further code will be executed If a function is running if a settimeout runs out first the function will be completed then the settimeout function will begin execution It triggers the browser timer after milllisecond with the given funciton.

2.6.2. JavaScript Engine

Thread of execution Memory Call stack Web Browser APIs Promises EventLoop, callback queue Callback queue will never dequeue until the global context will end

2.6.3. Callback Hell

Our response data is only available in the call back function

2.6.4. Event Loop

First check if the call stackk is empty else run it. If the call stack is empty then check and run the callback queues So until the stack is empty no callback queue will run ever
