3. Ansible

How to install zsh sudo apt install zsh chsh -s ’which zsh’ curl and install oh my zsh edit .zshrc add plugins={git zsh-autosuggestions} curl the plugin

To get a computer working we have to install all the libraries over and over again SSH keys are also needed to be setup

Ansible can be used here to setup computer fast TO setup ZSH using ansible

anisble create a yaml file fem.yml

hosts: localhost
become: true
pre_tasks:
 - name: Update Cache
   apt:
     update_cache: true
     tags:
     - node
     - zsh
tasks:
 - name: Install zsh
   apt: name=zsh
   tags:
   - zsh
 - name: Change Shell
   shell: chsh -s 'which zsh'
   tags:
   - zsh
 - name: Install oh my zsh
   shell: curl -L
   tags:
   - zsh
 - name: Install autosuggestions
   ansible.builtin.git:
      repo:"repo.git"
      dest: /path/to/zsh
   tags:
   - zsh
 - name: Update the zshrc
   shell: sed -i "s/(git/(git zsh-autosuggestions' ~/.zshrc"
   tags:
   - zsh
 - name: install node
   apt:
     name:['nodejs','npm']
   tags:
   - node
 - name: Install n
   npm:
     name: n
     global: true
   tags:
   - node
 - name: n 14
   shell: n 14
   tags:
   - node

This is better than bash install script as it allows for finer control to the system ed is an editor then sed is a stream ed editor Useful for log and repo

To look how to do something, google ansible git clone ansible-playbook fem.yml

Only execute stuff with the node tag ansible-playbook -t node fem.yml

ansible runs from top to bottom Try to anisible for everything

We can seperate this yml file node.yml zsh.yml and move all the zsh tasks to zsh.yaml and node tasks to node.yml and include this files in the main yaml files using tasks:

  • include: fem-tasks/zsh.yml
  • include: fem-tasks/node.yml

We can get ansible tasks from a remote repos ansible-pull -U url

11. Tmux

We have a tmux server which have multiple session Each sessions have one or more window Each windows have multiple panes sudo apt install tmux tmux This started a tmux server, New session with one window and a pane C-d - to remove pane without a pane a window is closed Without a window a session is closed without a session the server is closed Tmux is also scriptable to create what we want

tmux list-session List all the sessions tmux kill-server Kill the server tmux Create the tmux server tmux new-session Create a new tmux session tmux new-session -d Create a new tmux session detached mode tmux a attach the terminal prefix - ( Move to the next session prefix - ) Move to the previous session prefix - d Detach from the current session prefix - w See all the sessions with the windows C-d EOF Prefix key is C-b Next key is the command

Windows Group windows in sessions which have similar things prefix-C tmux new window Create new Window prefix - n Go to the next window prefix - p Go to the previous window prefix - 1-7 Go to the 1-7 window Rename a session tmux new-session -d -s "foo" Create a session with name tmux new-window -n "foo" create a new window with a name tmux new-session -d -s "foo" -n "Name" Create a session with name with a nae to the first window prefix - % helps to split a window into panes C - d to delete a pane we can create window or session for a particular directory tmux switch-client -t "foo" Go to the foo tmux new-window -n "foobar" [coomand] Create a new window with a name and start this command

12. FZF

It is quick fuzzy finder It helps writing the line to which matches the input the most C-r is used to FZF in the command history 

#!/user/bin/env bash
session =$(find ~ ~/personal/ ~/work/ -type d -mindepth 1 -maxdepth 1 | fzf)
session_name=$(basename "$session" | tr . _)

if !tmux has-session -t "${session_name" 2> /dev/null; then

    tmux new-session -s "$session_name" -c "$session" -d
fi

tmux switch-client -t "$session_name"

prefix-L to jump back a session zsh has Vi mode
