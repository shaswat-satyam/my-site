Bash

Extension .sh or nothing using chmod +x script name to add executable ./script.sh to run the script To identify a bash we use shebang #!/bin/bash Shebang tells the script which interpreter to use TO declare a variable var=value to Reference the variable use $ to differentiate with commands and variable echo $var Variables dont persist for logout and login

Strings with single quotes doesn’t use string templating It supports string templating by default \’ to escape characters to store the output of an command use $(cmd) This is a subshell which helps to execute a command in the background Few variables are declared by default known as environment variables Thery are captial $SHELL $USER Uppercase variables are used to reference environment variables They are generally avoided in scirpting programs env command to get all the environment variables

-eq is for equal to

count=12
if [ $count -eq 10 ]
then
    echo "Condition is True"
else
    echo "Condition is False"
fi

-ne is for not equal to

count=12
if [ $count -ne 10 ]
then
    echo "Condition is True"
else
    echo "Condition is False"
fi

-gt is for greater than

count=12
if [ $count -gt 10 ]
then
    echo "Condition is True"
else
    echo "Condition is False"
fi

-lt is for lesser than

count=12
if [ $count -lt 10 ]
then
    echo "Condition is True"
else
    echo "Condition is False"
fi
