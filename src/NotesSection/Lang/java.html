2. Exception Handling

An exception is a problem that arises during the executing of a program example -

  • Invalid file path
  • Invalid data Exception handling is a task to maintain normal flow of the program. Exception handling does not mean repairing an exception, its is rather defining alternative way to continue rest of the program normally

2.1. Exception handling in JAVA

2.1.1. Runtime Stack Mechanism

For every thread a runtime stack is created by JVM. Every method calls are maintained in a runtime stack.

2.1.2. Structure of Exception

Every Exception thrown is printed by JVM in the form Name: description Location(Stack Trace)

JVM checks if a method handling the exception else JVM handles the exception on its own.

2.1.3. Exception Hierarchy

Throwable class acts a root for entire Java Exception hierarchy. Exception and Error are inherited from throwable Exceptions are recoverable but Errors are not recoverable Errors can be caused by lack of system resources.

Exception are divided into

  1. Checked Exception

    These excceptions are checked by the compiler at the runtime

  2. Unchecked Exception

    These exceptions are not checked by the compiler.

2.2. Keywords for Exception handling

2.2.1. Try Catch Block

Contains the risky code within the try block and corresponding handling code inside the catch block. Inside the catck block we can print the exception

  • e.printStackTrace() gives the name:description and followed by stack trace
  • e.toString() gives the name:description No stack trace
  • e.getMessage() gives only the description

By default Exception handler using printStackTrace;

  1. Code
    try{
        int i = 100/0;
}catch(Exception e){
        System.out.print(e);
}

2.2.2. Try with multiple Catch Block

If try with multiple catch block is present then the order of catch blocks is very important and it should be from child to parent. First arithmetic exception will be checked and if it is not present then Exception class is checked. 

try{
        int i = 100/0;
} catch(ArithmeticException e){
        Sout(e);
} catch(Exception e){
        Sout(e);
}

2.2.3. Finally

Code in the finally block always executed, whether or not exception has occurred. Used to run any cleanup type statements that you want to execute. 

try{
        int i = 100/0;
}
catch(exception e){
        sout(e);
}
finally{
        sout("Program is done");
}

If the exception is handled

  1. Catch block is used
  2. Then the finally block is used

Else

  1. Finally block is used
  2. JVM handles the exception

2.2.4. Throw

It is used to explicitly throw an exception throw new exception(“exption”); Only throwable class can be throw.

    try{
        if(i<18){
                 throw new exception("Not adult");

2.2.5. Throws

Used to declare the exception, It provide information to the programmer that there may occur an exception. So during call of that method, programmer must use exception handling mechansim A non runtime exception can’t be propagated without throws Throws Keyword is also used to propagate checked exception. 3. Strings

3.1. String

String as a object that represent sequence of char values. Strings are defined in java.lang.String Strings in java is immutatable that is cannot be changed When a string is created

  1. JVM checks the String Pool
  2. A reference to the string is returned if the string is in the pool
  3. A new string is created if string is not in the pool

3.1.1. String s = “Hello”;

String s1 = "Hello"; s and s1 both have the reference to the same string in the StringPool This is the reason java Strings are immutable This helps in memory efficiency

3.1.2. String s1 = new String(“Hello”);

New keyword creates the object in the heap New memory is given in heap 

String s = "EEC"; // this creates a new string in the stringpool
s = s.concat("word"); // this creates a new string in the stringpool

Both the strings exist together in the pool.

3.2. String Comparison

Three methods for string comparison

3.2.1. equals()

Compares original content of the Strings compares the string for equality 

String s1 = "BBC";
String s2 = "BBC";
String s3 = new String("BBC");
String s4 = "Cringe";
s1.equals(s2); // True
s1.equals(s3); // True
s1.equals(s4); // False

3.2.2. == Operator

Compares references of the Strings compares the string for equality 

String s1 = "BBC";
String s2 = "BBC";
String s3 = new String("BBC");
String s4 = "Cringe";

s1 == s2; // True
s1 == s3; // False
s1 == s4; // False

3.2.3. compareTo()

Compare values lexicographically and returns an integer value if First string is less than equal to or greater than other string s1 is equal to s2 -> 0 s1 is greater to s2 -> positive s1 is lesser to s2 -> negative

3.3. StringBuffer

StringBuffer class are mutable StringBuffer sb = new StringBuffer(“Hello ”);

3.3.1. Important Methods

  1. Append()

    Adds a substring to the string at the end sb.append(“World”);

  2. Insert()

    Adds a substring to the string at a particular index sb.insert(1,“ nice ”);

  3. replace()

    sb.replace(1,3,“okay”) Replaces the start index to end index with the string

  4. delete()

    It deletes the substring from start to end index

  5. reverse()

    It reverses the string

3.3.2. Differences between string and string buffers

String StringBuffer
Immutable Mutable
Slow Fast
Consumes more memory because too many string concat Consumes less memory
Overrides equals() method of the object class Doesn’t Override equals() methods of object class

3.4. StringTokenizer

Defined in the java.util package Used to break a string into tokens import java.util.*; or java.util.StringTokenizer StringTokenizer s1 = new StringTokenizer(“Hello World”); default deliminator is whitespace

3.4.1. Constructors

  • StringTokenizer(String str)
  • StringTokenizer(String str, String deliminator)
  • StringTokenizer(String str, String deliminator, boolean returnValue) if returnValue is true deliminator is considered as tokens else deliminator is used only for separating tokens

3.4.2. Methods

  • boolean hasMoreTokens() checks availability of tokens
  • String nextToken() returns the next token
  • String nextToken(String deliminator) return next token on the basis of deliminator
  • boolean hasMoreElements() similar to has more tokens
  • int CountTokens() returns the total number of tokens

4. Applet

Applet is a java program that runs in a web browser. Applet can be runned only on a java enabled browesr. This is an old concept not used nowadays. Applet is a java class that extends the java.applet.Applet class. It has no Main() method. It has a public void paint() method. Applet are designed to be embedded within a HTML page. JVM is required to view an applet. Applet have strict security rules that are enforced by the web browser Sandbox Security. This is the rule that it can’t access the machine directly. appletviewer can be used to use an applet.

4.1. Life Cycle of Applet

Applet is similar to lifecycle of thread

  1. Applet initialized Use the init() method
  2. Applet Running Use the start() method
  3. Applet Stopped Use the stop() method then use start() again to restart the applet
  4. Applet Stopped Use the destroy() method

Suppose a.html is having a java code start() is called by the browser when the component of the applet is displayed. if component goes out of the view the stop method is called if the page is closed the destroy() method is called.

4.2. Creation of applet

Create two files java file and html code

4.2.1. Java file 

import java.applet.*;
import java.awt.*; // Graphics present in the awt package

public class app_image extend Applet{
        public void paint(Graphics g){
                g.drawString("My Applet",100,100);// Method to draw string at 100,100 coordinates
        }
}

Save as appimage.java javac appimage.java A file appimage.class will be created

4.2.2. HTML file

<html> <title> Image Example </title> <applet code = “appimage.class” width=400 height=400> </applet> </html>

Save as a.html Open html in java enabled web Browser Use appletviewer command appletviewer a.html

import java.applet.*;

public class app_image extend Applet{

                str = getParamter("name")
        }

        }
}

Save as appimage.java javac appimage.java A file appimage.class will be created

4.5. Java Sandbox Model

  • Security model for java
  • Provides restricted environment
  • Local code is trusted to have full access to vital system resources such as file system
  • Downloaded remote code is not trusted and can access only limited resources provided inside the sandbox
